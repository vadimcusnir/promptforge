
Reguli obligatorii — Background, Layout fix, Header/Footer, Login, Copy unic
1) Background — static, nemișcat, fără flicker (global + /dashboard)

Montează un singur root pentru background.

HTML:

{/* layout.tsx */}
<body className="app-shell">
  <div id="bg-root" aria-hidden="true"></div>
  {children}
</body>


CSS (Tailwind sau clasic):

:root { --header-h: 64px; --footer-h: 56px; }
html,body { height: 100%; overflow-x: hidden; background:#0b0b0b; }
.app-shell { position: relative; min-height: 100%; display:flex; flex-direction:column; }
#bg-root { position: fixed; inset: 0; z-index: 0; pointer-events: none; contain: strict; }
main, header, footer, [data-layer="ui"] { position: relative; z-index: 1; }
/* NICIODATĂ transform pe strămoșii elementelor fixed */
.app-shell, .app-shell *[data-layout-root] { transform: none !important; will-change: auto !important; }
/* Previne „bounce” și scroll chaining nedorit */
html, body { overscroll-behavior: none; }


Nu folosi background-attachment: fixed (buggy pe iOS). Ține background-ul ca DIV position: fixed.

În /dashboard, oprește orice animație de background.

.route-dashboard #bg-root * { animation: none !important; transition: none !important; }

2) /dashboard — layout rigid, jank-free

Grid fix, doar conținutul central are scroll:

.dash-grid {
  display: grid;
  grid-template:
    "header header" var(--header-h)
    "aside  main"   1fr
    "footer footer" var(--footer-h)
    / 260px 1fr;
  min-height: 100vh;
}
.dash-header { grid-area: header; position: sticky; top: 0; z-index: 100; }
.dash-aside  { grid-area: aside;  position: sticky; top: var(--header-h); height: calc(100vh - var(--header-h) - var(--footer-h)); overflow: auto; z-index: 80; }
.dash-main   { grid-area: main;   overflow: auto; /* singurul scroller */ }
.dash-footer { grid-area: footer; }


Interdicții stricte în /dashboard:

Fără transform pe .dash-grid, .dash-header, .dash-aside, .dash-main.

Fără parallax, fără blur pe scroll, fără “sticky” care schimbă dimensiuni (doar culoare/opacitate).

Fără “auto-resize” pe tabele; setează lățimi minime și virtualizează listele.

FPS guard: toate animațiile UI din /dashboard ≤ 120ms și doar pe opacitate/culori (nu pe layout).

3) Header — fix, fără schimbare de înălțime

Dimensiune constantă: --header-h: 64px desktop, 56px mobil. Niciun element din header nu schimbă înălțimea pe hover/scroll.

Z-index map:

background: 0

content: 1

header: 100

aside: 80

dropdowns: 200

modals/paywalls: 1100

toasts: 1200

Rezervă spațiul în conținut:

main { padding-top: var(--header-h); }
@supports(padding: max(0px)) {
  .dash-header { padding-top: env(safe-area-inset-top); }
}

4) Footer — stabil, niciodată peste conținut

Site (marketing): sticky footer cu flex-column, nu fixed:

.app-shell { display:flex; min-height:100%; }
main { flex: 1 1 auto; }
footer { height: var(--footer-h); margin-top: auto; }


/dashboard: footer în grid (vezi §2). Niciun “slide-up” sau “auto-hide”.

5) Login/Autentificare — o singură cale, fără SSR flicker

Route unică: /auth (email+pass + magic link). După login → redirect 302 la /dashboard.

Protejează server-side accesul:

Middleware: blochezi /dashboard fără sesiune → redirect /auth.

Fără mesaje variabile în înălțime: rezervă spațiu pentru erori (24px) ca să nu apară CLS.

SSR safety pentru background pe /auth: dacă folosești background, încarcă-l client-only (ssr:false).

6) Texte „o singură dată” — copiază dintr-o singură sursă (SSOT)

Creează lib/copy.ts (sau i18n) și scoate toate textele canonice acolo:

export const COPY = {
  brand: "PromptForge™ v3",
  tagline: "50 module. 7-D Parameter Engine. Export bundle.",
  cta_primary: "Try Generator",
  cta_pricing: "Pricing",
  legal_disclaimer: "Scorurile sunt estimative. Rezultatele pot varia.",
  paywall_pro: "Vezi scor real + Export PDF/JSON (Pro).",
  paywall_ent: "ZIP + API + Seats (Enterprise)."
} as const;


Interdicție: nu scrii aceste texte în componente. Importă din COPY.

Pentru mesaje dinamice (erori, validări), definește enum + map în lib/messages.ts. Nicio literă hard-coded în UI.

7) Constrângeri vizuale — fără salturi, fără „reflow”

Dimensiuni rezervate pentru toate imaginile/SVG-urile din header/footer (lățime/înălțime fixă).

Fără layout shift pe hover: schimbă doar opacitate/culoare; nu padding/margini.

Reduce-motion:

@media (prefers-reduced-motion: reduce) {
  * { animation-duration: 0.01ms !important; animation-iteration-count: 1 !important; transition-duration: 0.01ms !important; }
  #bg-root * { animation: none !important; }
}

8) Lintere & guardrails (ca să nu „scape” nimic)

stylelint regulă custom: interzice transform pe .app-shell, .dash-grid, header, footer:

// .stylelintrc.json (exemplu simplu)
{ "rules": { "plugin/no-transform-in-roots": [true, { "selectors": [".app-shell", ".dash-grid", "header", "footer"] }] } }


ESLint UI: interzice style={{ transform: ... }} în componente de layout („no-inline-transform-layout”).
